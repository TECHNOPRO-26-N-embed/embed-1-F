タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）

**↓ 自分のシステムを書いてください**

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON]
  ↓ 初期化完了───────────────────────────────────┌────────── (ボタン押下) ──────────────────────────→ [機能停止] ┐
[待機中] ─── (音と温度をセンサー検知) ───→ [DCモーター作動] ─── (5秒間温度が26℃以下 もしくは 60秒間音が40dB未満)    |
  ↑                                                               │               　　　　　　　　　   (ボタン押下)   
  └──────── リセット ──────────────────────────────────────────────┘───────────────────────────────────────────┘


```

※補足:ボタンを押下(一回目)で機能停止。再度ボタンを押下(二回目)する事で[待機中]に遷移

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 待機中 | センサー値を100msごとに監視する | DCモーター：停止 |
| DCモーター作動 | 温度・音の条件を監視しながら送風する | DCモーター：起動 |
| 機能停止 | ボタン押下後、モーターを止めて次のボタン押下を待つ | DCモーター：停止 |
|  |  |  |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:---------|:----------|:--------------|:----------|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
| センサー値の取得 | 100ms | **millis** | 他の処理を妨げないため |
| 音の継続時間の判定 | 60秒 | **millis** | 60秒間検知がなければファン停止。検知があれば再度60秒間監視を継続するため |
| 温度の継続時間の判定 | 5秒 | **millis** | 他のセンサー処理を止めないため |
| ボタン入力の監視 | 常時 | **millis** | 即時応答が必要なため |
| ファン制御 | 状態変化時 | **millis** | 状態に応じた即時制御が必要なため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
| チャタリング | ボタンを押したとき | 前回入力から50ms未満の入力を検出 | 50msデバウンス。確定入力のみ `buttonPressed` を更新 |
| センサー異常値 | センサーが誤読したとき | 値が仕様範囲外（例: 温度<0℃ or 温度>50℃、音ADC<0 or >1023） | 異常値を破棄し前回正常値を保持。`abnormalLogFlag` を立ててシリアルログ出力 |
| アナログノイズ | アナログ入力を読んだとき | 閾値付近で値が揺れる | 5点移動平均 + ヒステリシス（ON/OFFしきい値分離）で誤判定を防止 |
| 音の一時的な途切れ | 短時間だけ無音になるとき | 無検知時間を `millis()` で計測 | 60秒連続で無検知なら停止。途中で再検知したらタイマをリセットして監視継続 |
| センサー読取り失敗 | DHT11読み取りが失敗したとき | `isfinite()` で有限値か確認、または取得失敗コードを検出 | 今回周期の制御をスキップし、次周期で再試行。連続失敗時は安全側でファン停止 |
